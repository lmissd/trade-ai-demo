# 提交给 Codex 的启动提示词

请你在当前项目中开发一个“国际贸易 AI + 二维码库存 Demo 第一版”。

你必须先阅读：

1. `docs/PROJECT_DESIGN.md`
2. `docs/TODO.md`
3. `docs/PROJECT_MEMORY.md`

项目固定演示场景：

```text
中国采购 100 箱货
→ 发往赞比亚仓库
→ 上传合同和箱单
→ AI 识别信息
→ 生成 100 个二维码
→ 手机扫码入库
→ 销售出库 20 箱
→ 库存剩余 80 箱
→ 老板问 AI：这批货现在还有多少？
```

你的开发规则：

1. 一次只允许做 `docs/TODO.md` 中的一个大环节。
2. 不允许一次性完成全部功能。
3. 每次开始前，先读取设计方案、TODO 和项目记忆。
4. 找到第一个未完成的大环节，只实现这个环节。
5. 实现完成后必须自测。
6. 自测完成后必须更新 `docs/PROJECT_MEMORY.md`。
7. 自测完成后必须更新 `docs/TODO.md`，勾选已完成项。
8. 每个大环节完成后必须执行 git commit。
9. 每个大环节完成后必须 git push 到 GitHub。
10. 完成后必须停止，并向我汇报本环节完成内容、测试结果、commit 信息，然后等待我确认。
11. 未经我确认，不允许继续下一个环节。
12. 如果没有 GitHub remote，不要继续开发，先提示我提供 GitHub 仓库地址。
13. 不要把任何 API Key 写入前端代码。
14. AI 第一版可以先使用 mock 数据，保证 Demo 能跑通。
15. AI 不能直接修改数据库，所有写入必须经过后端接口和用户确认。

现在请开始执行阶段 0：项目初始化与规则固化。
