# PROJECT_MEMORY

## 项目名称

国际贸易 AI + 二维码库存 Demo 第一版

## 固定演示场景

```text
中国采购 100 箱货
→ 发往赞比亚仓库
→ 上传合同和箱单
→ AI 识别信息
→ 生成 100 个二维码
→ 手机扫码入库
→ 销售出库 20 箱
→ 库存剩余 80 箱
→ 老板问 AI：这批货现在还有多少？
```

## 开发规则

1. 每次只做一个大环节。
2. 完成一个环节后，必须更新本文件。
3. 完成一个环节后，必须更新 `docs/TODO.md`。
4. 完成一个环节后，必须 git commit。
5. 完成一个环节后，必须 git push。
6. 完成一个环节后，必须停止，等待用户确认。
7. 没有用户确认，不进入下一环节。
8. AI 不能直接修改数据库。
9. API Key 不能写入前端。
10. 第一版允许 AI mock，优先保证 Demo 跑通。
